# Agent Bootstrap

> **Purpose:** immutable entrypoint for every repository-backed agent session.
> **Audience:** the session agent, not the project owner.
> **Mutability:** do not edit this file in place.

A session starts from the full Git bundle and matching SHA-256 sidecar supplied by the project owner. This file is the only path the owner needs to name.

## Immediate guardrails

Before reading other repository material or attempting external acquisition:

- treat the supplied full bundle plus sidecar as the session's sole repository input authority;
- use only local Git to verify and reconstruct it in an isolated workspace;
- do not use the GitHub connector, GitHub API, `gh`, or network Git for clone, fetch, pull, push, or repository inspection;
- do not use `curl`, `wget`, or a presumed working DNS path to acquire repository or claim-bearing artifact bytes;
- use the Google Drive connector as the first transport and receipt-readback route when it is available;
- do not scan history, handoffs, stages, evidence, experiments, or exhaustive navigation during onboarding;
- do not begin implementation, research, or package construction before completing the mandatory read closure and reporting the onboarding certificate.

## Step 1 — establish the repository authority

1. Verify the external SHA-256 sidecar against the supplied bundle.
2. Run `git bundle verify`.
3. Reconstruct an isolated local repository from the bundle.
4. Record branch, HEAD, tree, bundle refs, and worktree cleanliness.
5. Do not contact or mutate the owner's canonical checkout.

A bundle or sidecar mismatch is fail-closed. Stop and report the mismatch without trying another transport path.

## Step 2 — complete the mandatory read closure

Read these files in order and in full:

1. [`docs/agent/PROJECT_MODEL.md`](docs/agent/PROJECT_MODEL.md)
2. [`docs/agent/SESSION_PROTOCOL.md`](docs/agent/SESSION_PROTOCOL.md)
3. [`docs/current/STATE.json`](docs/current/STATE.json)
4. [`docs/current/AGENT_TASK.json`](docs/current/AGENT_TASK.json)

Also run:

```bash
python3 tools/agent_onboard.py --root .
```

The script validates the immutable bootstrap contract and emits the mechanical portion of the onboarding certificate. It does not replace reading the mandatory files.

## Step 3 — load only the current task modules

Read every entry in `AGENT_TASK.json.required_reads` and `required_authorities`, respecting its declared scope and section boundary. Do not broaden the read set merely to gain general context.

Use exhaustive navigation only when a required path is missing or the current task discovers a concrete dependency not represented by the task manifest. Record every additional read and why it was necessary.

## Default exclusions

Unless `AGENT_TASK.json` explicitly selects them, do not read:

```text
docs/history/**
docs/handoff/**
docs/stages/**
legacy PROJECT_CONTEXT or PROJECT_ORIENTATION documents
unrelated docs/evidence/**
unrelated experiments/**
full roadmap documents outside the selected section
past chat transcripts
```

Generated human overviews such as `README.md`, `docs/CURRENT_CONTEXT.md`, and `docs/INDEX.md` are not substitutes for `STATE.json` and `AGENT_TASK.json` and are not part of mandatory agent onboarding.

## Step 4 — report the onboarding certificate

Before work begins, report:

```text
bundle SHA-256
bundle refs
branch / HEAD / tree
worktree cleanliness
bootstrap contract version
project-model revision
session-protocol revision
STATE revision
current epoch / gate / next action class
R, L, or T work classification
mandatory files read
task-specific files and exact sections read
additional files read and justification
default-excluded domains
claim and non-claim boundary
blockers and unresolved risks
```

If any required path, rule, section boundary, digest, or state/task relationship fails validation, stop at onboarding and report the exact failed invariant.
