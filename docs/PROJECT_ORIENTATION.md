# Project Orientation

This project adapts upstream CPython Android products for practical Termux CLI use while keeping launcher, native closure, provenance, archive identity, ownership, recovery, transition, consumer integration, and managed-Python distribution as separate authorities.

## Governing method

```text
understand -> reproduce -> measure -> compare -> design -> optimize
```

Complete independently audited Termux evidence outranks repository design, local reconstruction, static analysis, and chat memory. Never infer a broader claim from a narrower PASS.

## Current boundary

Stage 3-D is frozen through Gate 6. Stage 3-E Gate 2 is accepted by an external 117/117 re-audit of the preserved target archive, and Gate 3 freezes the managed-Python distribution contract.

Exact patch-version requests are authoritative. Minor and unspecified requests are conditional latest-patch surfaces and selected CPython 3.14.6 through the uv minor alias in both observed installation orders.

Gate 4 is active next. It validates a project-owned persistent managed root without using uv's default real managed directory, global executable links, shell edits, network acquisition, or uv patching.
