## Root cause

The Stage 3-A shell wrappers attempted to keep probe bytecode out of the runtime tree with:

```text
PYTHONDONTWRITEBYTECODE=1
PYTHONPYCACHEPREFIX=<results directory>
```

However, the fresh-process child probes were launched as:

```text
python -I -S -c ...
```

CPython isolated mode `-I` implies environment isolation behavior and ignores `PYTHON*` environment variables. The child interpreters therefore did not honor either bytecode-control environment variable.

The imported pure-Python dependencies were free to create `__pycache__` directories and `.pyc` files inside the promoted candidate prefix.

Reference:

```text
https://docs.python.org/3.14/using/cmdline.html
```

The relevant command-line contract is:

```text
-I  isolated mode; PYTHON* environment variables are ignored
-B  do not write .pyc files on source-module import
```

## Corrective change

The fresh-process probe child commands now explicitly carry `-B`:

```text
python -I -B -S -c ...
```

Applied to:

```text
experiments/stage3a-runtime-closure/probe-system-sonames.py
experiments/stage3a-runtime-closure/probe-extension-imports.py
```

This preserves the intended isolated import environment while making the no-bytecode-write contract explicit at the child interpreter command line.

The Phase 5 wrapper was also changed so that mutation or workflow failures still proceed through the machine-verdict writer. A failed gate should retain:

```text
results/termux/stage3b-promoted-closure/promoted-closure-verification.json
```

instead of exiting before the primary structured verdict is written.
