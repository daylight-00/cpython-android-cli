
They came from:


Frozen interpretation:

> These paths describe user installation destinations. Their presence in sysconfig metadata is not evidence that base interpreter execution depends on those directories.

No other existing `OTHER_ABSOLUTE` path remained after extractor-v2 correction.

## Build-prefix residue


Frozen interpretation:

> `/usr/local` paths remain as stale build/development metadata, not live runtime filesystem dependencies on the tested environment.

## Missing runtime-prefix paths

The 20 missing runtime-prefix records observed in triage came from inactive or generic installation schemes, including:

```text
nt
nt_venv
posix_home
```

The active default scheme was:

```text
posix_prefix
```

Frozen interpretation:

> Missing paths generated by inactive schemes must not be reported as defects in the active runtime layout.

Examples include Windows-style:

```text
<prefix>/Include
<prefix>/Lib
<prefix>/Scripts
```

and `posix_home` destinations such as:

```text
<prefix>/include/python
<prefix>/lib/python
```

which are scheme outputs, not necessarily required existing directories in the current runtime-only prefix.
