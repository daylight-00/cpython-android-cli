## File-entry count policy

The frozen Stage 3-A aggregate was:

```text
file_entry_count=3280
```

The workflow captures the complete candidate `files.tsv` and reports the aggregate delta, but does not make the raw count a semantic pass/fail gate. Generated runtime state can change the aggregate without changing the ELF closure or runtime product contract.

Any count difference is retained for row-level review before Phase 5 is frozen. The first Gate 2 attempt demonstrated why this distinction matters: native closure remained identical while probe-induced bytecode creation changed the candidate tree and correctly failed the separate mutation-control gate.
