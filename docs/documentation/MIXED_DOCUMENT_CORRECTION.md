# Mixed-document correction contract

> **Status:** frozen Phase 4 contract

Phase 4 separates stable project meaning, live coordinates, active plans, and historical snapshots without rewriting old authority bytes.

## Required outcomes

- one stable project guide contains no live gate or immediate action;
- one stable documentation-system guide contains no migration-phase status;
- one canonical summary plan contains no completion status;
- old orientation, contexts, summary roadmaps, and phase contracts remain byte-preserved snapshots or frozen authorities;
- the predecessor root README is preserved before the root entry is simplified;
- generated history and roadmap indexes expose lifecycle and successor metadata;
- current state selects the stable guide and canonical plan;
- no product, qualification, feature-selection, publication, or device claim changes.

Historical status words are not removed from original files. Their interpretation is normalized externally through registry and generated navigation.
