## Accepted conclusions

- the main near-term product should be upstream-derived from Python.org and BeeWare;
- Astral is the primary structural reference for product artifacts and metadata;
- Termux is a primary profile, not the runtime ABI;
- fresh-extraction relocation is required; post-mutation movability is not implied;
- uv is a preferred manager, not a runtime prerequisite;
- APK reuse remains a later consumer, not the initial architecture driver;
- API 36 is a required bounded comparison, not an automatic release floor;
- a full Astral-like source producer is deferred to Epoch 4.

## Superseded proposals

- direct promotion of the current component with full history into the clean product repository;
- making full source production an Epoch 3 prerequisite;
- treating the Note9 secondary run or emulator coverage as a blocker for recalibration;
- the former E2-P4 through E2-P7 canonical sequence.
